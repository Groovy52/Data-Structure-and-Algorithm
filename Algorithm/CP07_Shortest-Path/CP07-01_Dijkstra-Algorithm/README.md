# Dijkstra Algorithm

## 1. 다익스트라 알고리즘이란?

다익스트라(Dijkstra) 알고리즘은 **하나의 시작 정점에서 다른 모든 정점까지의 최단 거리**를 구하는 알고리즘입니다.  
그래프에서 간선의 가중치가 주어질 때, 시작점으로부터 각 정점까지 도달하는 최소 비용을 찾는 데 사용됩니다.

예를 들어 다음과 같은 상황에서 활용할 수 있습니다.

- 한 도시에서 다른 도시들까지의 최소 이동 거리 구하기
- 한 정점에서 각 정점까지의 최소 비용 구하기
- 네트워크에서 특정 노드로부터 다른 노드들까지의 최소 전송 시간 구하기

---

## 2. 다익스트라를 사용할 수 있는 조건

다익스트라 알고리즘은 다음 조건에서 사용할 수 있습니다.

- **간선의 가중치가 0 또는 양수**
- **음수 가중치가 존재하지 않아야 함**

즉, 거리, 시간, 비용처럼 일반적으로 음수가 될 수 없는 값을 다루는 그래프에서 적합합니다.

음수 가중치가 존재하는 경우에는 다익스트라가 올바르게 동작하지 않을 수 있으므로,  
그럴 때는 **벨만-포드(Bellman-Ford)** 알고리즘을 사용해야 합니다.

---

## 3. 핵심 아이디어

다익스트라의 핵심은 다음과 같습니다.

> 시작점에서 가장 가까운 정점부터 하나씩 최단 거리를 확정해 나간다.

이를 조금 더 풀어서 보면 다음과 같습니다.

1. 시작점의 거리를 0으로 둔다.
2. 현재까지 알려진 거리 중 가장 짧은 정점을 선택한다.
3. 그 정점을 거쳐 갈 수 있는 다른 정점들의 거리를 갱신한다.
4. 이 과정을 반복한다.

즉,  
**현재까지 발견한 최단 거리 후보들 중 가장 짧은 것부터 처리하는 방식**입니다.

---

## 4. 왜 우선순위 큐를 사용하는가?

다익스트라에서는 매번  
**가장 거리가 짧은 정점**을 빠르게 선택해야 합니다.

이를 위해 **우선순위 큐(Priority Queue)** 를 사용합니다.

우선순위 큐에는 보통 아래와 같은 형태로 데이터를 넣습니다.

- `(거리, 정점 번호)`

이렇게 하면 가장 거리가 짧은 정점이 먼저 꺼내집니다.  
즉, 다익스트라의 핵심인  
**“가장 가까운 정점부터 처리한다”**  
를 효율적으로 구현할 수 있습니다.

---

## 5. 그래프 표현 방식

다익스트라는 보통 **인접 리스트(Adjacency List)** 로 그래프를 표현합니다.

예를 들어 다음과 같은 방향 그래프가 있다고 가정하겠습니다.

- 1 → 2 (비용 2)
- 1 → 3 (비용 5)
- 2 → 3 (비용 1)
- 2 → 4 (비용 4)
- 3 → 4 (비용 2)

이를 인접 리스트로 표현하면 다음과 같습니다.

```python
graph = [
    [],
    [(2, 2), (3, 5)],
    [(3, 1), (4, 4)],
    [(4, 2)],
    []
]
```

## 6. 기본 코드 구현

위 코드는 다음 순서로 동작합니다.

1. 시작 정점의 거리를 0으로 설정한다.
2. 시작 정점을 우선순위 큐에 넣는다.
3. 가장 거리 값이 작은 정점을 꺼낸다.
4. 그 정점과 연결된 다른 정점들의 거리를 확인한다.
5. 더 짧은 경로를 발견하면 거리 값을 갱신하고 우선순위 큐에 넣는다.
6. 우선순위 큐가 빌 때까지 반복한다.

```python
from queue import PriorityQueue # 우선순위 큐를 사용하기 위해 불러옴

INF = 10**9 # 아직 도달하지 못한 정점의 거리를 아주 큰 값으로 초기화하기 위해 사용

def dijkstra(n, graph, start): # n: 정점의 개수, graph: 인접 리스트로 표현된 그래프, start: 시작 정점 
    
    dist = [INF] * (n + 1) # 시작 정점에서 i번 정점까지의 현재 최단 거리 후보
    selected = [False] * (n + 1) # i번 정점의 최단 거리가 이미 확정되었는지를 의미
    pq = PriorityQueue() # 우선순위 큐를 생성

    dist[start] = 0 # 시작점에서 시작점까지의 거리는 0
    pq.put((0, start)) # 우선순위 큐에 시작점을 넣음

    while not pq.empty(): # 우선순위 큐가 빌 때까지 반복, pq의 경우 while pq: 로만 둘 경우, pq가 비지 않았을 경우에도 True 처리되어 넘어갈 수 있음

        cur_dist, now = pq.get() # 우선순위 큐를 사용하기 때문에 가장 작은 거리 값을 가진 정점이 먼저 나오게 됨

        if selected[now]: # 이미 최단 거리가 확정된 정점이라면 건너뜀, 같은 정점이 우선순위 큐에 여러 번 들어가는 것을 막음 
            continue

        selected[now] = True

        for next_node, weight in graph[now]: # 현재 정점 now와 연결된 모든 인접 정점을 확인
            if not selected[next_node]:
                next_dist = cur_dist + weight # 현재 정점을 거쳐 다음 정점으로 가는 새로운 거리 후보를 계산

                if next_dist < dist[next_node]: # 기존에 알고 있던 거리보다 더 짧은 경로를 찾았다면 갱신
                    dist[next_node] = next_dist
                    pq.put((next_dist, next_node))

    return dist # 시작 정점에서 각 정점까지의 최단 거리 배열을 반환


n = 4
graph = [
    [],
    [(2, 2), (3, 5)],
    [(3, 1), (4, 4)],
    [(4, 2)],
    []
]

start = 1
result = dijkstra(n, graph, start)

for i in range(1, n + 1):
    if result[i] == INF:
        print(f"{i}번 정점: 도달 불가")
    else:
        print(f"{i}번 정점: {result[i]}")
```
