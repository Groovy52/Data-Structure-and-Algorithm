# 분할 정복 (Divide and Conquer)

분할 정복은 큰 문제를 **더 작은 같은 형태의 문제들로 나눈 뒤**, 각각을 해결하고, 그 결과를 **합쳐서** 원래 문제의 답을 구하는 알고리즘 설계 방식입니다.

핵심은 3단계입니다.

1. **Divide**: 문제를 작은 부분 문제로 나눈다  
2. **Conquer**: 부분 문제를 재귀적으로 해결한다  
3. **Combine**: 부분 문제들의 답을 합쳐 최종 답을 만든다  

예를 들어 배열의 합을 구한다고 하면, 배열을 반으로 나누고, 왼쪽 합과 오른쪽 합을 각각 구한 뒤 둘을 더할 수 있습니다.

---

## 1. 분할 정복의 직관

예를 들어 `[1, 2, 3, 4, 5, 6, 7, 8]`의 합을 구한다고 해보겠습니다.

전체를 한 번에 보지 않고,

- 왼쪽 `[1, 2, 3, 4]`
- 오른쪽 `[5, 6, 7, 8]`

로 나눕니다.

그리고 다시 각각을 반으로 나누어

- `[1, 2]`, `[3, 4]`, `[5, 6]`, `[7, 8]`

이런 식으로 계속 쪼갭니다.  
더 이상 나눌 수 없는 크기까지 가면, 그때부터 결과를 합쳐 올라옵니다.

이런 방식은 문제 크기를 빠르게 줄일 수 있어서 매우 자주 사용됩니다.

---

## 2. 분할 정복의 특징

분할 정복은 보통 이런 특징을 가집니다.

- 문제를 **비슷한 형태의 작은 문제**로 나눌 수 있다
- 작은 문제의 답으로 큰 문제의 답을 만들 수 있다
- 보통 **재귀**로 구현하는 경우가 많다
- 시간복잡도가 개선되는 경우가 많다

대표적인 예시는 다음과 같습니다.

- **이진 탐색**
- **병합 정렬**
- **퀵 정렬**
- **빠른 거듭제곱**
- **행렬 분할 연산**
- **FFT**

---

## 3. 분할 정복의 기본 구조

Python 기준의 가장 기본적인 형태는 보통 이렇게 생깁니다.

```python
def divide_and_conquer(problem):
    # 1. 종료 조건 (더 이상 나눌 수 없을 때)
    if problem_is_small_enough(problem):
        return solve_directly(problem)

    # 2. 분할
    subproblems = divide(problem)

    # 3. 정복
    subresults = []
    for sub in subproblems:
        subresults.append(divide_and_conquer(sub))

    # 4. 결합
    return combine(subresults)
