# Shortest Path

이 폴더는 **최단 경로(Shortest Path)** 알고리즘을 정리한 공간입니다.  
그래프에서 한 정점에서 다른 정점까지 이동할 때, **거리의 합이 최소가 되는 경로**를 찾는 알고리즘들을 모아두었습니다.

단일 출발지 최단 경로 문제는 다익스트라(digkstra) 알고리즘, 벨만-포드(Bellman-Ford) 알고리즘으로 해결할 수 있고, 모든 쌍 최단 경로 문제는 플로이드-워셜(Floyd-Warshall) 알고리즘으로 해결할 수 있습니다.

- [dijkstra](./CP10-01_Dijkstra-Algorithm/README.md)
- [bellman-ford](./CP10-02_Bellman-Ford-Algorithm/README.md)
- [floyd-warshall](./CP10-03_Floyd-Warshall-Algorithm/README.md)

각 알고리즘은 모두 최단 경로를 구하는 데 사용되지만,  
적용할 수 있는 조건과 사용 목적이 서로 다릅니다.

---

## 왜 여러 최단 경로 알고리즘이 필요한가?

최단 경로 문제는 항상 같은 방식으로 풀리지 않습니다.  
그래프의 조건에 따라 적절한 알고리즘이 달라집니다.

예를 들어,

- **간선 가중치가 모두 0 이상**이라면 → 다익스트라
- **음수 가중치가 존재할 수 있다면** → 벨만-포드
- **모든 정점 쌍 사이의 최단 거리**를 구하고 싶다면 → 플로이드-워셜

처럼 구분해서 사용합니다.

즉, 이 폴더는 단순히 여러 알고리즘을 나열한 것이 아니라,  
**상황에 따라 어떤 최단 경로 알고리즘을 선택해야 하는지 이해하기 위한 정리 공간**입니다.

# 1. Dijkstra Algorithm

## 1-1. 다익스트라 알고리즘이란?

다익스트라(Dijkstra) 알고리즘은  
**하나의 시작 정점에서 다른 모든 정점까지의 최단 거리**를 구하는 알고리즘입니다.

최단 경로 알고리즘 중에서도 가장 대표적이며,  
특히 **간선의 가중치가 음수가 없는 그래프**에서 매우 자주 사용됩니다.

예를 들어 다음과 같은 상황에서 활용할 수 있습니다.

- 한 도시에서 다른 도시들까지의 최소 이동 비용 구하기
- 출발 지점에서 각 지점까지의 최소 시간 구하기
- 네트워크에서 한 노드에서 다른 노드들까지의 최소 전송 비용 구하기
